Create a single-file web app: /root/pangkalan-map/server.py

Goal: Pangkalan Geo-Mapper — a dark-themed map dashboard for LPG pangkalan data served by Python http.server on port 5004, stdlib only, no Flask.

Database (read-only):
- Path: /root/second-brain/02 - LPG Work/database/pangkalan.db
- Table: pangkalan
- Columns of interest: id_registrasi (TEXT UNIQUE), nama_pangkalan, nama_pemilik, alamat, kota, kecamatan, no_hp_pemilik, qty_kontrak, nomor_nib, status, integrasi_map, latitude, longitude, provinsi, nama_agen, sold_to, wilayah

HTTP server requirements:
- Use http.server.HTTPServer and http.server.BaseHTTPRequestHandler (or SimpleHTTPRequestHandler subclass).
- Listen on 0.0.0.0:5004.
- Serve inline HTML/CSS/JS at GET /.
- JSON API routes:
  1. GET /api/pangkalan?status=&kota=&problem=&search=
     - Returns JSON: { "features": [...GeoJSON features...], "list": [...summary objects...], "count": N }
     - GeoJSON features: {"type":"Feature","geometry":{"type":"Point","coordinates":[lng,lat]},"properties":{ id_registrasi, nama_pangkalan, kota, status }}
     - list items: { id_registrasi, nama_pangkalan, kota, kecamatan, status }
     - Default query when no params (or empty): status='Active' AND kota='KOTA PONTIANAK'.
     - status filter: All / Active / Inactive. Empty or 'All' means no status filter.
     - kota filter: empty means no kota filter.
     - problem filter: comma-separated list of problem keys from checkboxes:
       - qty_over_3000 -> qty_kontrak > 3000
       - no_kota -> kota IS NULL OR trim(kota) = ''
       - no_nib -> nomor_nib IS NULL OR trim(nomor_nib) = ''
       - duplicate -> EXISTS another row with same (LOWER(TRIM(nama_pangkalan)), LOWER(TRIM(kota)), LOWER(TRIM(alamat))) (use case-insensitive)
       - no_coords -> latitude IS NULL OR longitude IS NULL OR latitude=0 OR longitude=0
       - none / empty -> no problem filter
       When multiple problems selected, OR them together.
     - search param: case-insensitive match against nama_pangkalan, nama_pemilik, alamat.
     - Only include rows with non-null lat/lng when building features (still include in list even without coords).
     - ORDER BY nama_pangkalan COLLATE NOCASE ASC.
     - Add LIMIT 500 to keep response small; if count > 500 add a message but still return 500 records.
  2. GET /api/pangkalan/detail?id_registrasi=XXX
     - Returns { "pangkalan": {...row...} } for the exact id_registrasi.
  3. GET /api/pangkalan/kota
     - Returns { "kota": ["KOTA PONTIANAK", "KABUPATEN ...", ...] } distinct non-empty kota ordered.
  4. GET /api/pangkalan/stats
     - Returns counts: { total, active, inactive, problematic }
     - problematic = count rows matching at least one of the 5 problem conditions.
